**Вариант 46879**

## 1. Постановка задачи

Провести интеграционное тестирование программы, осуществляющей вычисление системы функций:

```
        ┌  (((cos(x) + cos(x)) · sec(x)³) + cos(x)),                            x ≤ 0
f(x) = ─┤
        └  ((((log₂(x) · log₅(x) / log₂(x)) · log₂(x))²) + (log₂(x) / log₁₀(x))), x > 0
```

## 3. Анализ области определения системы

### 3.1. Ветка `x ≤ 0`

`f(x) = 2·cos(x) · sec³(x) + cos(x) = 2/cos²(x) + cos(x)`

Точки разрыва — нули `cos(x)`:
`x = −π/2, −3π/2, −5π/2, …`

При `x = 0` (граница) → `f(0) = 2/1 + 1 = 3`.

### 3.2. Ветка `x > 0`

`f(x) = (log₂(x) · log₅(x))² + log₂(x) / log₁₀(x)`

Точка разрыва: **`x = 1`** (`log₁₀(1) = 0` → деление на ноль).
ОДЗ: `x > 0, x ≠ 1`.

## 5. Стратегия интеграции — «по 1 модулю»

Применена **восходящая интеграция (bottom-up)**: каждый последующий тест добавляет к уже проверенным реальным модулям ещё один, заменяя оставшиеся mock-заглушками (`@Mock`/`@Spy` Mockito).

### 5.1. Уровни интеграции

| Уровень | Реальный модуль                  | Заглушки                    | Тест                          |
|---------|----------------------------------|-----------------------------|-------------------------------|
| L0      | `Sine`, `NaturalLogarithm`       | —                           | `SineTest`, `NaturalLogarithmTest` |
| L1      | `Cosine` (+ реальный `Sine`)     | `Sine` отдельно мокается    | `CosineIntegrationTest`       |
| L2      | `Secant`, `Cosecant`, `Tangent`, `Cotangent` | базовые мокаются | `*IntegrationTest` в `trig/integration` |
| L3      | `BaseNLogarithm`                 | `NaturalLogarithm` мокается | `LogarithmIntegrationTest`    |
| L4      | `EquationSystem` целиком         | все нижние модули мокаются  | `EquationSystemIntegrationTest` |

### 5.2. Что проверяется в `EquationSystemIntegrationTest`

1. **Маршрутизация по веткам**:
   - `x ≤ 0` → дёргаются только `Cosine`, `Secant` (`verifyNoInteractions` для логарифмов).
   - `x > 0` → дёргаются только `BaseNLogarithm(2/5/10)` (`verifyNoInteractions` для триг-модулей).
2. **Корректность вычисления**: параметризованный тест с CSV-эталоном `systemIT.csv` подменяет базовые модули заглушками, возвращающими значения `Math.cos`, `Math.log10`, `Math.log/Math.log(2)`, `Math.log/Math.log(5)` — итог собирается из проверенных «кубиков».

## 7. Утилита вывода в CSV

Класс `CSVGraphWriter` ([CSVGraphWriter.java](../src/main/java/ru/itmo/qa/lab2/util/CSVGraphWriter.java)):

```java
new CSVGraphWriter(any AbstractFunction, outputDir)
    .write(xFrom, xTo, step, precision);
```

- Принимает **любой модуль** (sin, cos, sec, csc, tan, cot, ln, log_n, EquationSystem) — параметр типа `AbstractFunction`.
- **Шаг наращивания `X` параметризуем** (`BigDecimal step` в методе `write`).
- Формат файла: header `x,y` и строки `x,f(x)` (разделитель — запятая, как разрешает задание).
- При разрыве функции (`ArithmeticException`) пишется пустая строка — для корректной отрисовки графика в `FunctionGraph`.

В `Main.java:58-69` все 11 модулей выводятся в свои `*.csv` файлы в каталоге `plots/`.
